normalizeColor

  • normalizeColor(color: string | number): number
    • Defined in src/normalize-color.ts:28

    This function will normalize RGB or RGBA colors from string to integers for Jimp to use. More on the Jimp GitHub Page

    However if already a number is passed, it will be returned as is. Furthermore a string will be tested against a regex, which will test if the string contains numbers. If the alpha channel is not provided, it will be added as 255 on the last position of the array (string is split). If the color is not a number or a string, it will be returned.

    // this is the initial rgb value.
    const rgb: string = '180, -20, 30';

    // will be transformed to colorchannels like so:
    const transformedRGBArr: string[] = [ '180', '20', '30', '255' ]

    // after that it will be clamped into a integer value like this:
    const rgbIntVal: number = 3021217535;

    Look into the function to understand it deeper.

    Parameters

    • color: string | number

      a string or number representing RGB or RGBA values: '180, -20, 30' or 3021217535 (both are the same)

    Returns number

    Will return a integer value representing a given RGB or RGBA string value.
